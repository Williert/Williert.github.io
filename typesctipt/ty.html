TypeScript

  • Ts 解决 Js 的类型问题。及 在定义的数据类型，与服务器响应的数据类型二者不一致，与我预想的数据类型的时候，容易发现错误。ts解决这个类型错误，但不会影响这个运行结果。 （静态类型检测器，类型推断）
  • 在编译的时候，及时发现错误，能有效减少运行时异常出现的机率。是一种严格的语法
  • 便于后期维护
  • ts代码先有编译器编译为js代码，再去js解析器再去执行。ts是js的超集，完全兼容js。

编译TS

ts需要编译为js后，借由js解析器运行在浏览器端。

  • 手动编译。

    tsc 文件名 — tsc ./helloworld.ts
  • vscode自动编译。

    先初始化 tsc -init 打开 tsconfig.json 配置文件
    • target : ts编译为什么样的形式（标准）
    • module : TypeScript文件中的module，采用何种方式实现
    • outDir : ts编译过后的js文件，存放在何处
    • strict : 严格模式

Ts 基础

Js数据类型分为两种，原始数据类型和复杂数据类型。ts的类型声明指定ts变量、参数的数据类型。如果没有定义数据类型，ts会自动推导数据类型

  • ts 会自动推导数据的类型，如果该变量的数据类型与定义的数据类型不一样的时候，就会报错。
  • 定义了什么类型的变量，就只能赋什么样的数据类型

undefined 与 null

还可以作为其他类型的子类型，可以赋值给其他数据类型的变量（严格模式除外）

数组 Array 与 对象 object

  • 如果数组的类型被定义了，里面的参数就只能是定义好了的数据类型的类型
    。如果数组的参数没有被定义，就不需要设置数组的数据类型
  • 数组的类型声明有两种方式，字面量和泛型
  • 对象的类型声明 ：

    比较特殊的是,object 表示非原始类型，就是除了number,string,boolean之外的类型
    对象的类型声明，通过构造函数去赋值不会报错，但字面量定义的方式会报错 （null，undefined，String，[],new 构造函数。不会报错）

任何类型 any

对于布尔值有两种表达形式，ture 和 1，但对于Ts而言，布尔值指的只有true,false

对于数组中的数据类型，数组中存储的数据类型是各种各样的,不能确定，此时可以 arr:any[ ] 来定义

  • any类型。数据类型无法确定的情况下 变量的类型是什么的时候，可以使用 any 来定义。但ts存在的意义就是对数据进行约束, any的使用，需要根据实际情况来使用

空值 void

  • 表示没有返回值的函数。虽然也能用来定义数据类型，但这种实际意义不大。

类型推论

没有明确的指定类型的话，ts会自动推断该数据的类型

  • 定义变量的时候，没有直接定义变量的数据类型，则定义类型为对应的类型
    • let num = 124
    • num = 数值类型的值
  • 仅声明变量，没有定义该变量的数据类型是，会定义该类型为any类型
    • let num;
    • num = 123 ; num=”123” ; num = true …

联合类型

表示取值可以为多种数据类型中的一种。Ts不确定一个联合类型的变量到底是什么数据类型的时候，能访问的属性和方法就只在联合类型的范围之内

  • 例如：我确定返回的数据是 boolean 或是 string ,这时候就不要使用any类型来定义，而是使用联合类型。（any类型是用来规避Ts的类型检查）
  • 如果对该变量再次赋值，Ts 会自动判断该类型
  • let num : number | boolean | number = true
  • 共有属性和方法 toString() …

对象的类型 - 接口

用来定义对象的类型。在面向对象语言中，接口（interface）是一个重要的概念，它是对行为的抽象，具体的实现需要类（class）去实现（impolement）.在 Ts 中，可以对对象的某一行为进行抽象，也可以对对象的数据类型进行描述。

  • 简单来讲，就是约束。约束对象中属性的数据类型。
  • 定义接口的时候，建议使用I作为前缀，例如（ IPerson ）
  • 实现接口时，不允许少一项接口的属性，也不能多一项属性

可选属性 ? ,不一定需要实现这个属性

任意属性，[propName:type] : type

某些情况下，允许一个接口有任意的属性。即，在实现接口的时候，不确定那个对象是否需要添加其他的属性，也不能确定这个属性的名称、类型是什么的时候。

  • [propName:string] ,确认了 propName 这个属性的是一个字符串
    • 值得一提的是，js对象的键值，都是以一个字符串的显示来表示，如果该 key 是一个数值类型，也会转化为一个字符串；如果是对象，会默认调用tostring进行转换（[object Object]）。换而言之，对象的键被自动转换为字符串。
  • 一旦定义了任意属性，那么就确定了属性和可选属性的类型都必须是它的子集
  • 一个接口只允许有一个 任意属性

只读属性

  • 在定义接口的时候，就定义属性只能可读，这样在实现的时候，从一开始就确定了其值不被修改，增加了安全性

枚举 enum

  • 枚举成员会被赋值为从0开始递增的数字，同时也会对枚举值到枚举名进行反向映射
  • 手动赋值，枚举会根据前面的值进行递增
  • 如果手动赋值与自动赋值的值，重复了，Ts不会报错，后面覆盖前面的值

常数项和计算所得项

  • 上面的直接赋值，就是 常数项。计算所得项，指的是 枚举的属性值是通过计算得出
  • 需要注意，计算所得项后面不能跟自动赋值项，因为在初始化时，还没有计算出值，但后面又需要依靠这个值进行递增

常数枚举 const enum

  • 普通枚举 与 常数枚举的区别在于 编译的时候，会被删除，并且不能包含计算成员

外部枚举 declare enum

  • 多用在声明文件.只会用于编译的时候检查，编译结果中会被删除

类 class

  • js中是通过构造函数来实现类的继承，在es6中迎来 class
  • Ts 除了实现了所有es6中类的功能外，还添加了一些新的用法

属性和方法

  • 与js不同的是，class的构造器中，必选指的参数的类型，才能允许使用。
  • 构造器形参默认类型 any ，建议添加上类型

继承

继承可以扩展现有的类

  • 类与类之间存在继承关系，通过 extends 继承
  • 子类 通过 super 调用 父类的属性和方法
  • 子类可以重新父类的方法

存储器

在TS中可以通过存储器（get,set），来截取对于对象成员的访问。也就是使用存储器去改变这个属性的赋值和行为

  • 存储器，可以控制对象成员的访问
  • get 只能读取值，不能修改值
  • set 可以修改值

静态成员 与 修饰符

  • 被 static 所修饰的 成员，可以直接通过类名访问，不需要通过类的实例访问
  • public 修饰的成员，都可以在任何地方被访问到，默认所有的成员都是 public
  • private 修饰的属性或方法是私有的，不能在声明它的类的外部访问
  • protected 修饰的属性或方法是受保护的，它和 private 类似，区别是它在子类中也是允许被访问的

初始化属性

修饰符修饰构造函数的形参，会创建并且初始化参数，而不需要先定义参数再赋值

抽象类

抽象类不能被实例化

  • 抽象类实际上是为子类服务的
  • 抽象类不能被实例，所以 constructor 没有定义的必要。但定义了也不会报错
  • 对于抽象方法，在定义时不能被实现。具体的实现需要子类操作
  • 抽象子类可以不实现抽象父类
  • 对于普通子类，继承抽象父类时，必须实现当中的抽象成员

类与接口

  • 接口与类联合使用，对类的一部分行为进行抽象。当多个类中的属性都一致时，可以抽取这部分
  • 可以实现多个接口
